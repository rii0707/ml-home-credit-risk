# Home Credit Default Risk — Production ML Project

End-to-end binary classification pipeline for the
[Kaggle Home Credit Default Risk](https://www.kaggle.com/c/home-credit-default-risk) competition,
refactored from a single notebook into a clean, modular codebase.

## Quick start

### 1. Create a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Download data

Either use **Download All** on the
[competition data page](https://www.kaggle.com/c/home-credit-default-risk/data)
and unzip into `data/raw/`, or run the helper (requires a
[Kaggle API token](https://www.kaggle.com/docs/api) and accepting the competition rules).

For **new API tokens** (`KGAT_...` from [API Tokens](https://www.kaggle.com/settings/api)): copy `config/access_token.example` to `config/access_token` and paste the token on one line (no quotes), or set `KAGGLE_API_TOKEN`. For **legacy** keys, use `~/.kaggle/kaggle.json` or `config/kaggle.json` from `kaggle.json.example`.

```bash
pip install kaggle
python scripts/download_home_credit_csvs.py
```

Required for the pipeline (eight CSVs):

- `application_train.csv`
- `application_test.csv`
- `bureau.csv`
- `bureau_balance.csv`
- `previous_application.csv`
- `installments_payments.csv`
- `POS_CASH_balance.csv`
- `credit_card_balance.csv`

The competition also ships `sample_submission.csv` and
`HomeCredit_columns_description.csv` (optional for this repo; see `data/raw/README.md`).

### 3. Run the full pipeline

```bash
cd src

# Step 1 — Load raw CSVs, reduce memory, save as parquet
python data_preprocessing.py

# Step 2 — Feature engineering + merge all tables
python feature_engineering.py

# Step 3 — Train all models (LR, RF, LightGBM + Optuna + ensemble)
python model_training.py

# Step 4 — Evaluation plots + error analysis
python evaluation.py
```

Each step can be run independently as long as its upstream artefacts exist.

## Module overview

| Module | Responsibility |
|---|---|
| `utils.py` | Shared constants (paths, seed, palette), logger factory, `reduce_mem_usage` |
| `data_preprocessing.py` | Load 8 raw CSVs, downcast dtypes, persist to parquet |
| `feature_engineering.py` | Application ratios, bureau/prev/installment/POS/CC aggregations, categorical encoding, final merge |
| `model_training.py` | `build_lr_pipeline`, `build_rf_pipeline`, `run_oof_cv`, `tune_lgbm` (Optuna), `find_best_blend_weights`, `train_all` |
| `evaluation.py` | ROC/PR curves, threshold analysis, confusion matrix, error-type breakdown, feature importance plot |

### Scripts (`scripts/`)

| Script | Responsibility |
|---|---|
| `download_home_credit_csvs.py` | Download competition CSVs into `data/raw/` (Kaggle API; see `config/` and `scripts/README.md`) |

## Key design decisions

- **Parquet checkpoints** — each step writes parquet; restarting doesn't re-run upstream work.
- **Pure functions** — every feature function takes a DataFrame in and returns one out (no global state).
- **Stratified OOF CV** — all OOF predictions use the same `StratifiedKFold` split for fair comparison.
- **`scale_pos_weight`** — class imbalance (~8 % positive rate) handled natively in LightGBM.
- **Optuna** — optional; falls back gracefully to `DEFAULT_LGB_PARAMS` if not installed.
- **Memory optimisation** — `reduce_mem_usage` downcasts dtypes, cutting RAM usage ~40–60 %.

---

## Reproducing results

All random operations use `SEED = 42` defined in `utils.py`.
Set `N_FOLDS = 5` (also in `utils.py`) to change cross-validation folds.

---

## Extending the project

- Add new side-table aggregations in `feature_engineering.py` following the existing pattern.
- Register new models in `model_training.py` by calling `run_oof_cv` and adding to `oof_preds_map`.
- Adjust `FN_COST / FP_COST` in `evaluation.py` for domain-specific threshold selection.
- Uncomment the FastAPI block in `app/app.py` to expose a prediction endpoint.
